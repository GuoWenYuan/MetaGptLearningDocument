**修改后的策划案**

1. **功能概述**
   - 玩家获取幻兽坐骑后解锁该坐骑的骑乘功能。
   - 玩家可以设置偏好坐骑，骑乘时系统将根据偏好随机召唤。
   - 组队跟随时，系统自动召唤骑乘或偏好坐骑。
   - 玩家可以通过主界面快捷键快速召唤坐骑。

2. **功能说明**
   - **功能入口**
     - 在主界面添加“坐骑”按钮，点击后显示坐骑功能界面。
     - 功能开启条件应该在功能开启表中配置，并在未满足条件时提供相应的用户反馈。

   - **坐骑默认界面**
     - 增加一个筛选功能，让玩家可以根据品质、已解锁等条件筛选坐骑。
     - 坐骑列表的排序逻辑需要优化，确保用户界面友好性。

   - **单行信息**
     - 显示坐骑名称和骑乘状态，对未解锁的坐骑进行明显的视觉区分。

   - **技能**
     - 技能展示应该清晰，并且确保拖拽技能到快捷栏的功能流畅。

   - **骑乘设置**
     - 为了避免玩家混淆，需要在界面上清晰地标识出坐骑的地面和飞行状态。

   - **动作查看**
     - 提供一个预览功能，让玩家可以查看坐骑的动作和特效。

   - **骑乘**
     - 需要提供更明确的用户反馈，例如当玩家成功骑乘或卸下坐骑时。

   - **坐骑传记**
     - 应该有一个专门的界面展示坐骑的背景故事和详细信息。

   - **偏好设置**
     - 提供一个更直观的界面，让玩家可以轻松设置和调整偏好坐骑。

   - **限时坐骑**
     - 对于限时坐骑，需要有清晰的倒计时提示和到期处理机制。

   - **解锁、骑乘状态变更提示**
     - 当玩家解锁新坐骑或获得新的骑乘状态时，应该有明显的界面提示。

   - **骑乘规则**
     - 需要确保骑乘规则简单明了，避免玩家混淆。

   - **自动骑乘逻辑**
     - 自动骑乘应该是一个可选项，并且玩家可以在设置中开启或关闭。

   - **选择偏好坐骑**
     - 提供一个更加直观的方法，让玩家可以快速选择偏好坐骑。

   - **其他策划支持**
     - 需要确保数值、关卡和配置表的内容与坐骑系统兼容。

   - **关联内容**
     - 死亡复活、交易、摧毁和丢弃等机制需要与坐骑系统紧密结合。
